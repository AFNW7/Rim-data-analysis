# 任务面板

## 当前目标

把项目从“数据扫描骨架”推进到“可配置白板小人战斗分析器”的第一版。

## V1 决策

按最新范围约定：

- `V1` 只支持 `RimWorld` 原版与已安装 DLC
- `V1` 优先完成“原版武器、护具、基础战斗分析”的可用流程
- `Mod` 导入不是取消，而是延后为 V1 之后的扩展功能
- 只有在原版分析流程已经可实际使用、可验证、可分享后，才开始扩展 Mod 支持

## 当前范围

第一版聚焦以下能力：

- 定义攻击方与防守方白板小人
- 支持基础种族基线：默认 `RimWorld` 原版智人
- 支持部分原版特性对战斗的影响
- 支持衣物/护甲叠穿合法性检查
- 支持指定身体区域上的多层护甲结算
- 支持基础远程与近战命中率、单次期望伤害、DPS 计算
- 支持从 JSON 场景文件运行分析

## 暂不纳入

- 完整解析原版与 Mod 的全部武器、服装、身体部位 Def
- 全身命中部位随机分布与器官穿透细节
- Combat Extended 等重写战斗系统的 Mod
- 战斗逐 tick 模拟与蒙特卡洛模拟

## 里程碑

### 里程碑 1：战斗分析最小可用版

状态：进行中

交付内容：

- `analyze-scenario` CLI
- 白板小人、武器、护具、修正器数据模型
- 多层护甲期望结算
- 远程/近战基础期望输出
- 示例场景
- 单元测试

### 里程碑 2：原版 Def 接入

状态：待开始

交付内容：

- 从原版 XML 读取武器与护具
- 生成标准化装备目录
- 把装备目录接入场景分析器

### 里程碑 3：V1 原版分析流程闭环

状态：进行中

交付内容：

- 原版武器/护具目录导出
- 基础武器 vs 护具分析表
- 一套明确的 CLI 使用流程
- 一份可直接展示在 GitHub 的样例输出
- 场景库、模板继承、横向对比表

### 里程碑 4：Mod Def 接入

状态：待开始

交付内容：

- 从 Workshop 与本地 Mod 解析武器与护具
- 兼容常见 Def 继承与 Patch 后结构
- 支持“原版 + 指定 Mod 集”分析
