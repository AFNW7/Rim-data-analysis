# 任务板

## 当前阶段目标

把项目从“功能已经可用，但结构和产品化仍然混乱”的状态，推进到“单一桌面入口、普通用户主流程清晰、可以继续稳定迭代”的状态。

## 执行原则

- 先解决会持续放大维护成本的问题，再做新功能。
- 先解决普通用户无法顺畅启动和使用的问题，再做高阶能力。
- 优先做可以为后续所有迭代降成本的改动。

## P0 本周必须完成

### P0-1 桌面端入口收口

状态：已完成

目标：

- 只保留 `desktop_user_app.py` 作为实际桌面实现。
- 将 `desktop_app.py` 和 `desktop_app_ui.py` 收缩为兼容包装层。
- 明确 CLI 和 README 中唯一推荐的桌面启动方式。

验收标准：

- `rim-analysis app` 仍可正常启动桌面端。
- 仓库内不再存在两套并行维护的桌面主实现。
- 旧模块导入不报错，但不会再承载实际业务逻辑。

### P0-2 首次启动与数据导入引导

状态：已完成

目标：

- 把“导入原版数据”整理成更像向导的流程。
- 降低用户对 `Data` 目录、Steam 路径概念的理解门槛。
- 所有关键失败提示给出下一步操作建议。

验收标准：

- 新用户首次启动后知道先做什么。
- 路径错误时，提示中包含“应该选到哪一层目录”。
- 成功导入后，界面明确告知下一步去哪里继续操作。

### P0-3 普通用户主流程收口

状态：已完成

目标：

- 统一“数据导入 -> 人物创建 -> 场景设计 -> 结果对比”的主流程。
- 减少页面内分叉操作和开发者视角文案。
- 把关键动作按钮位置和命名再收敛一轮。

验收标准：

- 普通用户不需要理解 JSON、defName、内部文件路径。
- 每个页面都能看出“当前这一步做什么”。
- 页面主要空间优先留给可操作区域，而不是说明性内容。

## P1 结构优化

### P1-1 拆分超大桌面类

状态：已完成

目标：

- 按页面拆分人物创建、场景设计、结果对比、数据导入、资源管理。
- 抽离公共组件、列表交互工具和共享状态。

验收标准：

- 单个页面逻辑不再全部堆在一个 2000+ 行类里。
- 页面改动不需要频繁触碰不相关页面代码。

### P1-2 拆分数据层职责

状态：已完成

目标：

- 将用户存储、静态选项、目录索引、预览计算、场景分析拼装拆成独立模块。

验收标准：

- `user_app_data.py` 不再同时承担常量表、仓库存储和分析拼装三类职责。

### P1-3 GUI 交互一致性整理

状态：已完成

目标：

- 统一单击、多选、双击、批量添加、批量删除的交互规则。
- 降低“某处支持多选，某处又只允许单选”的理解成本。

验收标准：

- 用户在不同页面面对列表时，交互预期一致。
- 所有限制型操作都有明确提示，而不是试错后才知道。

## P2 产品化增强

### P2-1 结果页从表格提升为结论页

### P2-2 正式应用数据目录

状态：已完成

目标：

- 将 `artifacts/app-state` 迁移到更合理的应用数据目录方案。
- 为后续安装包和版本升级做准备。

### P2-3 资源管理增强

状态：已完成

目标：

- 增加重复场景清理、预设重命名、分类管理等能力。
