# 项目路线图

## 阶段 1：元数据扫描

目标：让项目先“可运行、可分享、可扩展”。

交付内容：

- CLI 扫描器
- 路径自动发现
- `About.xml` 解析
- Def 统计
- JSON/CSV 导出
- 最小测试

建议分析问题：

- 原版、DLC、Mod 的结构差异
- 常见 Mod 的 Def 类型分布
- 哪些 Mod 更依赖 XML，哪些更依赖 DLL

## 阶段 1.5：白板小人战斗分析器

目标：让项目具备“武器与护具输出/承伤分析”的可运行核心。

交付内容：

- 白板小人数据模型
- 武器与护具数据模型
- 特性/器官/自定义修正器
- 护具叠穿检查
- 多层护甲结算
- 远程/近战命中率与 DPS 估算
- JSON 场景分析 CLI

建议分析问题：

- 某武器打某护甲套装的期望输出是多少
- 某套护具面对 Sharp/Blunt 的承伤差异有多大
- 技能和特性对同一武器收益如何变化
- 近战武器对重甲目标的真实效率如何

## V1 范围说明

当前 `V1` 范围已收敛为：

- 只支持原版 `Core + DLC`
- 先实现原版武器、护具、战斗分析的可用工作流
- `Mod` 导入作为后续扩展阶段处理

## 阶段 2：配置与加载顺序分析

目标：引入玩家真实使用场景。

交付内容：

- `ModsConfig.xml` 解析
- 包依赖与加载顺序图
- 缺失依赖、潜在冲突提示

建议分析问题：

- 当前 Mod 列表的依赖闭包
- 高频一起启用的 Mod 组合
- 大型 Mod 列表的结构复杂度

## 阶段 3：存档分析

目标：从“包结构分析”进入“游戏运行结果分析”。

交付内容：

- 存档 XML 解析器
- 世界、地图、殖民者、科技、事件、物品统计
- 可重复生成的分析报告

建议分析问题：

- 殖民地发展速度
- 科技树推进情况
- 角色技能与装备分布
- 不同 Mod 组合下的内容膨胀程度

## 阶段 4：报告与展示

目标：把项目变成适合 GitHub 展示的成果物。

交付内容：

- Notebook 示例
- 静态 HTML 报告
- 截图与样例输出
- 一份可复现的数据分析案例
