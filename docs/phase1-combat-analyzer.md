# 第一版战斗分析器设计

## 1. 目标

第一版的目标不是完整复刻整个 `RimWorld` 战斗系统，而是建立一个可扩展、可验证的“输出/承伤分析核心”。

它要支持用户：

- 创建攻击方白板小人
- 创建防守方白板小人
- 给双方挂特性、器官/能力修正、自定义修正
- 给防守方穿装备并检查叠穿是否合法
- 给攻击方装备武器
- 计算在指定距离、指定身体区域上的命中率、期望伤害、DPS、减伤率

## 2. 数据对象

### 2.1 白板小人

白板小人默认使用原版智人基线：

- `shooting_skill`
- `melee_skill`
- `sight`
- `manipulation`
- `moving`
- `traits`
- `custom modifiers`
- `apparel`

第一版只建模和战斗结果强相关的能力值，不包含需求、情绪、意识之外的大量非战斗属性。

### 2.2 特性与修正

第一版先支持这几类高价值修正：

- `tough`：最终承伤倍率
- `careful_shooter`：提高射击精度，增加瞄准时间
- `trigger_happy`：降低射击精度，缩短瞄准时间
- `brawler`：提高近战命中
- `nimble`：提高近战闪避

同时保留通用修正器接口，用于表达：

- 器官替换
- 伤病影响
- 任务/角色增益
- 用户手工输入的偏移

### 2.3 护具

每件护具至少包含：

- 名称
- 来源
- 所在层
- 覆盖的身体区域
- `Sharp / Blunt / Heat` 护甲值

第一版的叠穿规则：

- 若两件护具覆盖了相同身体区域
- 且它们占用了相同 layer
- 则视为冲突，不能同时穿戴

### 2.4 武器

武器按远程与近战分别建模，但共享这几个核心字段：

- 名称
- 攻击模式
- 基础伤害
- 伤害类型
- 护甲穿透
- 攻击周期参数

远程武器额外包含：

- `accuracy_close`
- `accuracy_short`
- `accuracy_medium`
- `accuracy_long`
- `warmup`
- `cooldown`
- `burst shot count`
- `burst shot interval`

近战武器额外包含：

- `cooldown`

## 3. 计算流程

### 3.1 远程命中率

第一版使用下列近似流程：

1. 由射击技能与特性得到 `shooting accuracy per tile`
2. 根据距离得到 `pawn accuracy ^ distance`
3. 乘上武器对应距离档位的精度
4. 乘上环境/自定义命中系数
5. 再乘上 `1 - cover_block_chance` 得到最终命中目标的概率

说明：

- 第一版按 `close(<=3) / short(<=12) / medium(<=25) / long(>25)` 四档取武器精度
- 第一版不做 stray shot、误伤、拦截体、弹道落点模拟

### 3.2 近战命中率

第一版按原版 wiki 给出的命中/闪避曲线计算：

- 攻击方得到 `melee hit chance`
- 防守方得到 `melee dodge chance`
- 最终命中率 = `melee hit chance * (1 - dodge chance)`

### 3.3 护甲结算

第一版按原版护甲三段式规则计算期望值：

1. `effective armor = armor - armor penetration`
2. 每层护甲都有三种结果：
   - 无伤弹开
   - 伤害减半
   - 护甲无效
3. 若仍有剩余伤害，则进入下一层
4. `Sharp` 在任意一层被缓解后，若还有最终伤害，则最终伤害类型转为 `Blunt`
5. 最后再乘以最终承伤倍率，例如 `tough`

### 3.4 输出指标

第一版输出：

- 叠穿是否合法
- 对目标身体区域生效的护甲层
- 每层的有效护甲与局部概率
- 命中率
- 单发命中期望伤害
- 单次攻击期望伤害
- 理论 DPS
- 实战期望 DPS
- 减伤率

## 4. 第一版限制

- 全身命中部位分布尚未纳入
- 未建模盾带、爆炸 AOE、强制偏移、友伤
- 未解析原版 XML 武器/衣物 Def
- 目前更适合“给定装备组合”的战斗数学分析
