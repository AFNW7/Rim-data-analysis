# 狙击 VS 靶70 计算流程

本文档记录的是当前项目代码实际使用的公式和结果，用于核对现版本实现，不代表 RimWorld 原版的完整底层机制。

## 1. 场景输入

- 场景名：`狙击 VS 靶70`
- 攻击方：`狙击`
- 防守方：`靶70`
- 距离：`40 格`
- 场景命中率倍率：`100% = 1.0`
- 掩体阻挡：`0`
- 攻击方射击等级：`14`
- 攻击方特性与额外增益：`无`
- 武器：`大师 狙击步枪/vanilla`
- 防守方护甲：`全身护甲 70%`

## 2. 武器数据

### 2.1 原版导入值

`Gun_SniperRifle` 当前导入到项目后的基础数据为：

- 基础伤害：`25.0`
- 基础护甲穿透：`37.5`
- 基础瞄准时间：`3.5s`
- 基础冷却时间：`1.5s`
- Burst 数：`1`
- 贴近精度：`0.50`
- 近距离精度：`0.70`
- 中距离精度：`0.88`
- 远距离精度：`0.90`

### 2.2 大师品质修正后的参战值

当前项目对远程武器使用的大师品质倍率是：

- 远程伤害倍率：`1.25`
- 远程护穿倍率：`1.25`
- 远程精度倍率：`1.35`
- 品质不影响 warmup / cooldown

代入后：

- 最终伤害：`25.0 x 1.25 = 31.25`
- 最终护甲穿透：`37.5 x 1.25 = 46.875`
- 最终贴近精度：`0.50 x 1.35 = 0.675`
- 最终近距离精度：`0.70 x 1.35 = 0.945`
- 最终中距离精度：`0.88 x 1.35 = 1.188`，按项目当前上限裁到 `1.0`
- 最终远距离精度：`0.90 x 1.35 = 1.215`，按项目当前上限裁到 `1.0`
- 最终瞄准时间：`3.5s`
- 最终冷却时间：`1.5s`

## 3. 命中率计算

### 3.1 人物每格精度

当前项目用 `SHOOTING_ACCURACY_BY_STYLE["standard"]` 查表。

- 射击等级：`14`
- 特性修正：`0`
- 视力修正：`0`
- 操作修正：`0`
- 额外 modifier：`0`
- 最终射击精度 stat：`14`
- 对应每格精度：`0.98`

### 3.2 40 格时的人物距离衰减

先不算武器，只看人物部分：

`人物命中保留率 = 0.98 ^ 40 = 0.44570040395095073`

即：

- 14 级射手在 40 格、先不算武器时，命中保留率为 `44.5700%`

### 3.3 最终命中率

40 格在当前项目中属于 `long` 档。

- 武器远距离精度：`1.0`
- 场景命中率倍率：`1.0`
- 掩体阻挡：`0`

当前项目的远程命中公式是：

`raw_hit = (人物每格精度 ^ 距离) x 武器该距离精度 x 场景命中率倍率`

`final_hit = raw_hit x (1 - 掩体阻挡)`

代入：

`raw_hit = (0.98 ^ 40) x 1.0 x 1.0 = 0.44570040395095073`

`final_hit = 0.44570040395095073 x (1 - 0) = 0.44570040395095073`

即：

- 最终命中率：`44.5700%`

## 4. 射击周期计算

当前场景没有额外瞄准或冷却修正，因此：

- 瞄准时间倍率：`1.0`
- 远程冷却倍率：`1.0`
- Burst 追加间隔：`(1 - 1) x 0 = 0`

当前项目的周期公式是：

`射击周期 = warmup x 瞄准时间倍率 + cooldown x 冷却倍率 + burst 追加间隔`

代入：

`射击周期 = 3.5 x 1.0 + 1.5 x 1.0 + 0 = 5.0s`

即：

- 射击周期：`5.00s`

## 5. 护甲与穿透结算

### 5.1 有效护穿

当前场景里攻击方没有额外护穿 modifier，因此：

`有效护穿 = 武器护穿 x 护穿倍率 = 46.875 x 1.0 = 46.875`

### 5.2 有效护甲

防守方是 `全身护甲 70%`，当前项目直接当作一层全身锐钝热都为 `70` 的 QuickArmor。

`有效护甲 = 护甲值 - 有效护穿 = 70 - 46.875 = 23.125`

### 5.3 当前项目的三段护甲概率

当前项目公式：

- `deflect = effective_armor / 200`
- `full_damage = 1 - min(effective_armor, 100) / 100`
- `half_damage = 1 - deflect - full_damage`

代入 `23.125`：

- 偏转概率：`23.125 / 200 = 0.115625`
- 全伤通过概率：`1 - 23.125 / 100 = 0.76875`
- 半伤概率：`1 - 0.115625 - 0.76875 = 0.115625`

即：

- `11.5625%` 完全弹开
- `11.5625%` 造成半伤
- `76.8750%` 造成全伤

### 5.4 命中后的期望伤害

单发原始伤害是 `31.25`。

当前项目对 `Sharp` 伤害的半伤分支会记为 `Blunt` 结果，因此这一层的结果分布是：

- `Sharp`：`31.25 x 0.76875 = 24.0234375`
- `Blunt`：`15.625 x 0.115625 = 1.806640625`
- 被弹开：`0 x 0.115625 = 0`

所以：

`命中后的期望伤害 = 24.0234375 + 1.806640625 = 25.830078125`

对应减伤率：

`护甲减伤率 = 1 - 25.830078125 / 31.25 = 0.1734375`

即：

- 命中后期望伤害：`25.830078125`
- 护甲减伤率：`17.34375%`
- 伤害保留率：`82.65625%`

如果你要把它当作当前项目里的“穿甲效率”近似理解，这里更接近：

- 对 70 护甲目标的平均伤害保留率：`82.65625%`

## 6. DPS 计算

### 6.1 理论 DPS

当前项目公式：

`理论 DPS = 单轮原始伤害 / 射击周期`

本场景是单发武器，所以单轮原始伤害就是 `31.25`。

代入：

`理论 DPS = 31.25 / 5.0 = 6.25`

### 6.2 期望 DPS

当前项目公式：

`单轮期望伤害 = 最终命中率 x 命中后期望伤害`

`期望 DPS = 单轮期望伤害 / 射击周期`

代入：

`单轮期望伤害 = 0.44570040395095073 x 25.830078125 = 11.512476254397116`

`期望 DPS = 11.512476254397116 / 5.0 = 2.3024952508794234`

另外，当前项目还会给出一个实现率：

`DPS 实现率 = 期望 DPS / 理论 DPS = 2.3024952508794234 / 6.25 = 0.36839924014070774`

即：

- 理论 DPS：`6.25`
- 期望 DPS：`2.3024952508794234`
- DPS 实现率：`36.839924014070774%`

- 命中后期望伤害：`25.830078125`
- 理论 DPS：`6.25`
